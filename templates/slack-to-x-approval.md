# Slack-to-X: Approval Template

**Purpose:** A clean template Mike reviews in under 2 minutes. Shows the original message, the transformed post, scoring rationale, quality gate results, and decision options.

---

## Template

```
# Slack-to-X: Approval Request
**Date:** [Date]
**Speed:** [Time-sensitive / Normal]
**Flagged by:** Dakota

---

## The Original (Mike's Slack Message)
**Channel:** #[channel]
**Date:** [Date and time]

> [Full text of Mike's original Slack message]

---

## The Transform
**Character count:** [X]
**Format:** [Single post / Thread / QRT of @handle]

---
[The transformed X post]
---

**Now/Near/Far:**
- Now: [What this enters]
- Near: [What this reveals]
- Far: [What this implies - not stated in post]

**Quality Gate:** [X/10] - All 5 gates passed / [Note any concerns]

**Sensitivity:** [Clean / Flagged items that were redacted]

---

## Mike's Decision

[ ] **APPROVE** - Post as-is (or with Mike's inline edits)
[ ] **KILL** - Not worth posting
[ ] **HOLD** - Save for better timing

**If link needed:** [What goes in the first reply]

---
```

---

## Usage Notes

- Route this template to Mike via Slack DM.
- Mike responds with one of three options: Approve, Kill, Hold.
- If Mike edits inline, the edited version is the final version.
- For approved posts, Mike posts manually from phone (prototype phase).
- Dakota logs the decision for weekly metrics tracking (approval rate, time from flag to live, engagement data).
